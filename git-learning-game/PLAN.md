# 개발 계획 및 일정

## 목표
- 터미널 기반 Git 학습 게임에서 협업 시나리오(가상 협력자) 중심으로 cherry-pick, rebase 등 실무 난관을 자연스럽게 익히게 한다.
- 문제 실패 시 힌트/해답 조회를 트리거로 동일 스테이지를 한 번 더 반복 진행하여 핵심 기능을 확실히 체득하도록 한다.

## 마일스톤(6주 기준)
### 1주차: 기획/설계
- 스테이지 구성 원칙 정의: 난이도 곡선, 실무 시나리오, 평가 기준
- 협업 시뮬레이션 설계: 가상 협력자 이벤트, 알림, 충돌 유도 규칙
- 힌트/해답 정책 확정: 사용 시 동일 스테이지 재도전 플로우

### 2주차: 스테이지/콘텐츠 설계
- 핵심 커맨드 목록 확정: cherry-pick, rebase, revert, reset, reflog 등
- 스테이지 템플릿 설계: 목표/제약/검증 로직/힌트/해답
- 실패/재도전 UX 정의: 반복 스테이지 진입, 진행도 계산 방식

### 3주차: 백엔드 구현
- 스테이지 검증 로직 구현(명령 기록, repo 상태 검사)
- 힌트/해답 조회 API 및 재도전 정책 구현
- 협업 시나리오 트리거(가상 커밋, 충돌 이벤트) 구현

### 4주차: 프론트엔드 구현
- 터미널 UI에 힌트/해답 버튼 및 재도전 안내 추가
- 협업 이벤트 UI(알림, 가상 협력자 메시지) 구현
- 스테이지 진행/재도전 상태 표시 개선

### 5주차: 난이도 조정/밸런싱
- 스테이지 성공률 측정/분석
- 힌트 사용 시 재도전 흐름 점검
- cherry-pick/rebase 고난도 시나리오 조정

### 6주차: 안정화/테스트
- 회귀 테스트(스테이지 검증, 협업 이벤트, 재도전 흐름)
- 문서 업데이트(README, 기여 가이드)
- 데모 시나리오 정리

## 세부 작업(핵심 정책)
- 힌트/해답 조회 시 동일 스테이지 1회 재진행을 강제한다.
- 재진행은 새로운 세션으로 기록하되, 학습 누적 지표에 반영한다.
- 해답 사용 여부를 결과에 명시하여 자가 학습 루프를 강화한다.
